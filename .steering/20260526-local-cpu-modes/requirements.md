# ローカル対戦・CPU対戦機能追加

## 目的

ガイスターにローカル対戦モードとCPU対戦モードを追加する。

## 要件

### 1. ローカル対戦モード

#### 基本仕様
- 同じ端末で2人が交互にプレイ
- オフライン動作（インターネット不要）
- 状態管理はReact stateのみ（localStorage不要）

#### ゲームフロー
1. 初期配置フェーズ
   - Player1が配置 → 配置完了ボタン
   - 画面を隠す（「Player2の番です」表示）
   - Player2が配置 → 配置完了ボタン
   
2. ゲームプレイフェーズ
   - 現在のターンのプレイヤーだけが駒の種類を見れる
   - 移動完了後、「ターン終了」ボタン
   - 画面を隠す（「相手のターンです」表示）
   - 「準備完了」ボタンで次のプレイヤーに交代

#### UI要件
- ターン交代画面：大きく「Player X の番です」表示
- 「準備完了」ボタンをクリックするまで盤面を隠す
- 自分のターンでは自分の駒の種類（good/bad）が見える
- 相手のターンでは何も見えない

### 2. CPU対戦モード

#### 基本仕様
- プレイヤー vs CPU
- CPUのターンは自動実行
- CPUの思考時間：1秒程度

#### AI仕様（初期実装）

**レベル1: ランダムAI**
- 合法手からランダムに選択
- 勝利条件は理解している

**レベル2: 基本戦略AI（将来拡張）**
- 相手のgood駒を優先的に取る
- 自分のbad駒を相手に取らせる
- 脱出を狙う

#### ゲームフロー
1. 初期配置
   - プレイヤーが配置
   - CPUは自動配置（ランダムまたは定石）
   
2. ゲームプレイ
   - プレイヤーのターン：通常通り
   - CPUのターン：1秒待機 → 自動で移動

### 3. モード選択画面の更新

現在の `/games/geister` ページを更新：

```
- オンライン対戦（実装済み）
- ローカル対戦（新規）
- CPU対戦（新規）
```

## 非機能要件

- オンライン対戦のコードに影響を与えない
- 既存のゲームエンジン（lib/games/geister/engine.ts）を再利用
- レスポンシブ対応（スマホでも快適）

## 実装優先度

1. ローカル対戦（比較的簡単）
2. CPU対戦（AIロジックが必要）
